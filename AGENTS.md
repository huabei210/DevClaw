# Project Notes

- `/history` 默认语义是查看完整历史，不应默认只显示最近几轮，也不应截断正文；单条消息放不下时应拆成多条发送。
- `/history <count>` 表示查看最近 N 轮历史，仍然保持正文完整，不做内容截断。
- `/history s` 是简洁预览模式，可以压缩正文长度，但应覆盖完整历史范围，主要用于快速浏览。
- 只要改了用户可见的命令、交互行为或默认语义，必须同步更新项目 README 文档，不能只改代码不改文档。
- `tsx watch <entry>` 在当前项目里不会可靠覆盖所有依赖文件；开发态热更新必须显式用 `--include` 把相关源码目录加入监听范围。
- `formatTranscriptTimestamp()` 的用户可见时间格式必须包含秒；测试或文档看到 `MM-DD HH:mm:ss` 时，不要再误改回只显示到分钟。
- `/cancel` 和 `/stop` 的产品语义不能只停留在“取消队列”；如果当前 thread 已在运行，必须尽力终止对应 assistant 活动进程及其子进程树，并同步更新 README。
- `/threads` 的数量参数不能再被硬编码上限 20 截断；用户写多少就应尽量返回多少，超出实际总数时返回全部。带多个连续命令时，同一会话的文本命令响应必须串行发送，避免把旧 `/dashboard` 输出串到新的 `/threads` 结果里。
